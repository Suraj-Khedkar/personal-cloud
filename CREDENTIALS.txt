  Remote:      https://<tailscale-ip>   (after Tailscale setup)

  Old URL (deprecated): http://192.168.1.4:8080

## iPhone Nextcloud App

  1. Update server URL to: https://192.168.1.4
  2. Accept the self-signed certificate warning once
  3. For remote: run scripts/add-tailscale-domain.sh, then use https://<tailscale-ip>

## Security checklist (do in Nextcloud web UI)

  [ ] Settings → Security → Enable Two-Factor Authentication (TOTP app)
  [ ] Users → Create a non-admin user for your iPhone
  [ ] iPhone app → use app-specific password (Settings → Security → Devices & sessions)
  [ ] Disable admin login on phone — use regular user account

## Commands

  Start:           cd ~/personal-cloud && docker compose up -d
  Stop:            cd ~/personal-cloud && docker compose down
  Apply proxy:     ./scripts/apply-nextcloud-proxy.sh 192.168.1.4
  Add Tailscale:   ./scripts/add-tailscale-domain.sh
  Regenerate SSL:  ./scripts/regenerate-ssl.sh 192.168.1.4 [tailscale-ip]

## Tailscale setup (remote access)

  1. brew install --cask tailscale
  2. Open Tailscale app → Log in
  3. Run: ./scripts/add-tailscale-domain.sh
  4. Optional: ./scripts/regenerate-ssl.sh 192.168.1.4 $(tailscale ip -4)
  5. docker compose restart nginx
  6. Install Tailscale on iPhone (same account)
  7. iPhone Nextcloud URL: https://<tailscale-ip>

## Migration to Raspberry Pi or old phone

  Full step-by-step guide: ~/personal-cloud/MIGRATION.md

  Quick summary:
  - Copy ~/personal-cloud/ to new device
  - Mount Cloud drive at /mnt/cloud (Pi) or internal storage (phone)
  - Update .env: CLOUD_DATA_PATH and NC_HOST
  - Run regenerate-ssl.sh and apply-nextcloud-proxy.sh on new device
  - Point iPhone to new Tailscale IP
